# Contributing to Gimba Goyo Personal Website

Thank you for your interest in contributing! While this is a personal website, suggestions and improvements are welcome.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:

- Check existing issues first
- Create a new issue with clear description
- Include screenshots if relevant
- Specify browser and device information

### Suggesting Improvements

Suggestions are welcome for:

- Performance optimizations
- Accessibility improvements
- UI/UX enhancements
- Code quality improvements
- Documentation updates

### Submitting Changes

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test thoroughly
5. Submit a pull request

## Development Setup

### Prerequisites

- Node.js 18+
- pnpm (or npm/yarn)
- Git

### Getting Started

1. Clone your fork:
```bash
git clone https://github.com/YOUR_USERNAME/gimbagoyo-website.git
cd gimbagoyo-website
```

2. Install dependencies:
```bash
pnpm install
```

3. Start development server:
```bash
pnpm dev
```

4. Open http://localhost:5173 in your browser

## Coding Standards

### JavaScript

- Use modern ES6+ syntax
- Follow existing code style
- Write clear, self-documenting code
- Add comments for complex logic
- Use meaningful variable names

### HTML/CSS

- Use semantic HTML5 elements
- Follow mobile-first responsive design
- Maintain consistent styling
- Ensure accessibility
- Test across browsers

### Vite Configuration

- Keep configuration simple
- Document any changes
- Test build process

## Testing

Before submitting:

1. Run development server and test locally
2. Build for production: `pnpm build`
3. Preview production build: `pnpm preview`
4. Run linter: `pnpm lint`
5. Test in multiple browsers
6. Test on different screen sizes

## Commit Messages

Write clear commit messages:

```
Add animation to CV section
Fix mobile navigation issue
Update profile information
Improve accessibility features
Optimize images for performance
```

## Pull Request Process

1. Update documentation if needed
2. Ensure all tests pass
3. Write clear PR description
4. Be responsive to feedback

### PR Description Should Include

- Summary of changes
- Reason for changes
- Screenshots (if UI changes)
- Testing performed
